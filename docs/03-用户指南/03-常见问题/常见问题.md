# 常见问题解答 (FAQ)

## 📖 概述

本文档收集了 GitHub 中文翻译插件(GitHub_i18n)的常见问题和解答，帮助用户快速解决使用中遇到的问题。

## 🔍 常见问题

## 安装与配置

## # Q: 哪些浏览器支持 GitHub_i18n 插件

* *A:** 所有支持用户脚本的现代浏览器都支持，包括：

- Google Chrome
- Microsoft Edge
- Mozilla Firefox
- Apple Safari
- Opera

使用前需安装用户脚本管理器，如 Tampermonkey、Violentmonkey 等。

## # Q: 如何安装 GitHub_i18n 插件

* *A:** 有两种方法：

1. **推荐方法**：从项目发布页面点击安装链接，自动安装到用户脚本管理器
2. **手动方法**：下载`GitHub_zh-CN.user.js`文件，导入到用户脚本管理器中

详细安装步骤请参考[使用说明](使用说明.md)。

## # Q: 插件是否需要额外配置

* *A:** 默认情况下，插件安装后即可使用，无需额外配置。高级用户可以通过修改源代码中的配置选项来调整插件行为。

## 使用问题

## # Q: 插件不生效怎么办

* *A:** 请按照以下步骤排查：

1. 确认用户脚本管理器已正确安装并启用
2. 确认 GitHub_i18n 脚本已启用
3. 刷新 GitHub 页面
4. 检查浏览器控制台是否有错误信息
5. 尝试重新安装最新版本

## # Q: 翻译不完整或不准确怎么办

* *A:**

- 当前版本可能只翻译了部分高频 UI 元素
- 新发布的 GitHub 功能可能尚未添加到翻译词典中
- 您可以通过项目的 Issues 页面报告未翻译或翻译不准确的内容

## # Q: 插件会影响 GitHub 的功能吗

* *A:** 插件设计为只替换文本内容，不会修改页面结构和功能，因此不会影响 GitHub 的正常使用。如果遇到功能异常，请在 Issues 中报告。

## # Q: 插件会影响页面加载速度吗

* *A:** 插件已经实现了多种性能优化：

- 使用翻译缓存减少重复翻译
- 采用节流和防抖机制控制翻译频率
- 优先翻译可见区域内容
- 这些优化确保插件对页面加载速度的影响最小化

## 更新与维护

## # Q: 插件如何更新

* *A:** 插件内置自动更新检查功能：

- 每次访问 GitHub 时，会在后台检查新版本
- 发现新版本时会提示用户更新
- 您也可以手动从发布页面下载最新版本重新安装

## # Q: 更新插件后翻译内容是否会丢失

* *A:** 不会，翻译词典是内置在插件中的，更新不会影响翻译功能。

## # Q: 如何获取插件的更新日志

* *A:** 您可以在以下位置查看更新日志：

- 项目的[版本历史](../10-版本历史/README.md)
- GitHub 仓库的 Release 页面

## 隐私与安全

## # Q: 插件会收集用户数据吗

* *A:** 不会。GitHub_i18n 是一个本地运行的用户脚本，不会收集或发送任何用户数据。所有翻译操作都在本地浏览器中完成。

## # Q: 插件会影响 GitHub 的安全吗

* *A:** 插件仅执行文本替换操作，不会修改页面功能或引入安全风险。源代码完全开源，您可以检查所有实现细节。

## 其他问题

## # Q: 如何报告翻译错误或请求添加新的翻译

* *A:** 请通过以下方式报告：

1. 在 GitHub 仓库中创建新的 Issue
2. 详细描述问题，包括未翻译的文本和所在页面
3. 如果可能，提供截图或页面 URL

## # Q: 如何为项目贡献代码或翻译

* *A:** 欢迎通过 Pull Request 方式贡献：

1. Fork 项目仓库
2. 进行修改
3. 创建 Pull Request 提交更改
   详细的贡献指南请参考[开发者文档](../07-开发者文档/贡献指南.md)。

## # Q: 插件支持除中文外的其他语言吗

* *A:** 当前版本仅支持中文翻译。如果您希望添加其他语言支持，可以通过项目 Issues 提出请求或直接贡献翻译词典。

## 📝 补充说明

如果您遇到的问题未在本 FAQ 中列出，请在项目 GitHub 仓库的 Issues 页面提交问题报告，我们会尽快回复并解决。
