# 06-用户手册

本文档提供了 GitHub 中文翻译插件(GitHub_i18n)的全面用户手册，帮助用户了解如何充分利用该插件的各项功能，优化 GitHub 使用体验。

## 🏃 快速开始

### 1. 安装插件

**前置条件**：

- 浏览器（Chrome、Firefox、Edge、Safari 等）
- 用户脚本管理器（推荐使用 Tampermonkey）

**安装步骤**：

1. 安装用户脚本管理器（如 Tampermonkey）
2. 从项目发布页面安装 GitHub 中文翻译插件
3. 确认插件已启用

### 2. 基本使用

安装完成后，插件会在您访问 GitHub 网站时自动激活，无需额外操作：

- 插件会自动将 GitHub 界面元素翻译成中文
- 支持 GitHub 的所有主要页面和功能
- 保持页面布局和功能完整性

## 📚 功能详解

### 1. 自动翻译功能

**功能描述**：插件会自动检测并翻译 GitHub 界面中的文本内容，包括导航栏、按钮、表单、菜单等各种 UI 元素。

**工作原理**：使用内置的翻译词典，通过文本匹配和替换的方式实现翻译，不会修改页面结构。

**翻译范围**：

- 导航和菜单
- 按钮和表单
- 提示信息和标签
- 帮助文本和说明

### 2. 实时监控更新

**功能描述**：插件会持续监控页面的 DOM 变化和路由切换，确保新加载的内容也能得到及时翻译。

**工作原理**：使用 MutationObserver 监听 DOM 变化，配合事件监听捕获路由切换。

**支持场景**：

- AJAX 动态加载内容
- 下拉菜单和模态框
- SPA 路由切换
- 实时通知和动态更新

### 3. 版本更新检查

**功能描述**：插件会定期检查是否有新版本发布，及时提示用户更新。

**工作原理**：通过与 GitHub API 通信，比较本地版本和远程版本信息。

**更新流程**：

1. 检测到新版本时显示更新提示
2. 用户点击更新按钮
3. 自动安装新版本
4. 刷新页面应用更新

## ⚙️ 配置指南

### 1. 基本配置

您可以通过修改插件源代码中的`CONFIG`对象来调整基本配置：

```javascript
const CONFIG = {
  // 是否启用自动更新检查
  updateCheck: true,
  // 版本检查的间隔时间（毫秒）
  versionCheckInterval: 24 * 60 * 60 * 1000, // 默认24小时
  // 是否启用翻译
  enableTranslation: true,
  // 是否翻译代码内容
  translateCode: false,
  // 是否翻译链接文本
  translateLinks: true,
};
```

### 2. 性能优化配置

对于性能有更高要求的用户，可以调整以下性能相关参数：

```javascript
const CONFIG = {
  // 防抖延迟时间（毫秒）
  debounceDelay: 200,
  // 节流时间（毫秒）
  throttleTime: 1000,
  // 是否启用翻译缓存
  enableCache: true,
  // 缓存大小限制
  cacheSizeLimit: 5000,
};
```

### 3. 页面模式配置

不同类型的 GitHub 页面可以有不同的翻译策略：

```javascript
const CONFIG = {
  // 页面模式配置
  pageModes: {
    // 仓库页面模式
    repository: { enabled: true, priority: 10 },
    // Issues页面模式
    issues: { enabled: true, priority: 9 },
    // Pull Requests页面模式
    pullRequests: { enabled: true, priority: 9 },
    // 全局页面模式
    global: { enabled: true, priority: 1 },
  },
};
```

## ❓ 常见问题

### 1. 为什么有些内容没有被翻译？

**解答**：

- 当前版本可能只翻译了部分高频 UI 元素
- 新发布的 GitHub 功能可能尚未添加到翻译词典中
- 某些特殊格式的内容（如代码块）默认不翻译

### 2. 插件会影响 GitHub 的性能吗？

**解答**：
插件已经实现了多项性能优化措施：

- 使用翻译缓存减少重复翻译
- 采用节流和防抖机制控制翻译频率
- 优先翻译可见区域内容
  这些优化确保了插件对页面性能的影响最小化。

### 3. 如何报告翻译错误或请求新翻译？

**解答**：
您可以通过项目的 GitHub Issues 页面报告翻译错误或请求添加新的翻译内容。

## 💡 最佳实践

### 1. 保持更新

定期更新插件以获得最新的翻译内容和功能改进。

根据您的设备性能和使用习惯，调整合适的性能参数：

如果发现未翻译或翻译不准确的内容，请及时报告，帮助改进翻译质量。
