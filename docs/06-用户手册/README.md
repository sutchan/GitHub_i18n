## 快速入门

### 插件概述

GitHub 中文翻译插件是一个强大的浏览器用户脚本，能够自动将 GitHub 网站的界面翻译成中文，提供更友好的中文使用体验。插件通过监听页面变化，实时翻译新加载的内容，确保整个 GitHub 浏览体验的连贯性。

### 首次使用

安装插件后，打开 GitHub 网站，您会立即看到界面已经变成中文。插件会自动运行，无需任何手动操作。

**注意**：部分内容可能需要页面完全加载后才能翻译完成。

### 主要功能

- 自动翻译 GitHub 界面元素
- 实时监控页面变化，自动翻译新增内容
- 支持自定义翻译开关
- 提供翻译质量反馈机制
- 自动检查更新

## 基本使用方法

### 访问 GitHub 网站

1. 在已安装插件的浏览器中打开 GitHub
2. 刷新页面，界面将自动翻译成中文
3. 浏览各个功能页面，体验中文界面

### 翻译状态指示

- 插件运行时，用户脚本管理器图标会显示活动状态
- 页面加载和翻译过程中，部分内容可能短暂显示英文
- 翻译完成后，整个页面将以中文显示

### 导航 GitHub 功能

插件不会改变 GitHub 的任何功能和导航结构，您可以按照平时的习惯使用 GitHub，只是界面语言变为中文：

- **首页**: 显示个性化推荐和活动
- **探索**: 发现热门仓库和开发者
- **Pull requests**: 查看和管理拉取请求
- **Issues**: 跟踪和讨论问题
- **Marketplace**: 浏览 GitHub 市场
- **Gist**: 创建和分享代码片段

## 高级功能使用

### 翻译控制

插件提供多种方式控制翻译行为：

- **临时禁用**: 点击用户脚本管理器图标，暂停插件运行
- **部分翻译**: 某些特殊内容可能不会被翻译，保持原始语言
- **刷新翻译**: 刷新页面可以重新触发翻译过程

### 翻译反馈

如果您发现翻译不准确或缺失，可以：

1. 提交 issue 报告问题
2. 提供正确的翻译建议
3. 参与翻译词典的改进

### 性能优化

为了获得最佳性能体验：

- 保持插件更新到最新版本
- 定期清理浏览器缓存
- 对于大型仓库页面，可能需要更长时间完成翻译

## 常见使用场景

### 浏览仓库

1. 访问任何 GitHub 仓库
2. 查看中文界面的代码、问题、Wiki 等内容
3. 使用中文搜索仓库内容

### 参与协作

1. 在中文界面中创建 Issues
2. 提交 Pull Request 并使用中文进行描述
3. 参与中文评论和讨论

### 学习和探索

1. 利用中文界面更容易理解 GitHub 的各项功能
2. 通过中文描述发现感兴趣的项目和资源
3. 参考中文文档和说明

## 使用技巧和最佳实践

### 提高翻译准确性

- 如果发现翻译不准确，可以复制英文原文搜索更准确的信息
- 对于专业术语，建议同时关注英文原文和中文翻译

### 提升使用效率

- 使用 GitHub 快捷键（在中文界面下仍然有效）
- 利用浏览器书签快速访问常用的 GitHub 页面
- 结合其他浏览器扩展使用，提升整体工作效率

### 常见问题解决

- **翻译不完整**: 刷新页面或清除缓存
- **性能问题**: 关闭不必要的浏览器标签或扩展
- **翻译错误**: 提交反馈帮助改进
