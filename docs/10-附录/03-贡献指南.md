# 贡献指南

## 📝 目录说明

本文档详细说明如何为 GitHub 中文翻译插件项目贡献代码、翻译、文档或其他资源，帮助项目不断改进和发展。

## 🤝 贡献类型

### 代码贡献
- 修复 bug
- 实现新功能
- 性能优化
- 代码重构
- 安全增强

### 翻译贡献
- 完善现有翻译
- 添加新的翻译词条
- 优化翻译质量
- 统一术语翻译

### 文档贡献
- 更新用户文档
- 编写技术文档
- 修正文档错误

### 测试贡献
- 提交测试用例
- 报告问题和缺陷
- 提供使用反馈
- 进行兼容性测试

## 🔄 贡献流程

### 代码贡献流程

1. **Fork 项目仓库**
   - 在 GitHub 上 Fork 项目到您的个人账号

2. **克隆仓库**
   ```bash
   git clone https://github.com/YOUR_USERNAME/GitHub_i18n.git
   cd GitHub_i18n
   ```

3. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   # 或者修复 bug 时
   git checkout -b fix/your-bug-fix
   ```

4. **安装依赖**
   ```bash
   npm install
   ```

5. **进行开发**
   - 实现功能或修复 bug
   - 遵循项目的代码规范
   - 添加相应的测试

6. **运行测试**
   ```bash
   npm test
   ```

7. **代码风格检查**
   ```bash
   npm run lint
   ```

8. **提交代码**
   - 使用清晰的提交信息
   - 遵循 Conventional Commits 规范
   ```bash
   git add .
   git commit -m "feat: add new feature"
   # 或
   git commit -m "fix: resolve bug issue"
   ```

9. **推送到远程仓库**
   ```bash
   git push origin feature/your-feature-name
   ```

10. **创建 Pull Request**
    - 在 GitHub 上创建新的 Pull Request
    - 清晰描述您的更改和目的
    - 引用相关的 Issue

11. **等待审核**
    - 项目维护者会审核您的代码
    - 根据反馈进行必要的修改

12. **合并代码**
    - 审核通过后，代码将被合并到主分支

### 翻译贡献流程

1. **Fork 并克隆仓库** (同代码贡献流程 1-3)

2. **修改翻译文件**
   - 翻译文件位于 `src/dictionaries/` 目录
   - 可以添加新的翻译词条或修改现有翻译

3. **测试翻译效果**
   - 构建项目：`npm run build`
   - 在浏览器中测试翻译效果

4. **提交代码并创建 Pull Request** (同代码贡献流程 8-12)

### 文档贡献流程

1. **Fork 并克隆仓库** (同代码贡献流程 1-3)

2. **修改或创建文档**
   - 文档位于 `docs/` 目录
   - 遵循项目的文档格式规范

3. **提交代码并创建 Pull Request** (同代码贡献流程 8-12)

## 📋 开发规范

### 代码规范

- 使用 ES6+ 语法
- 遵循项目现有的代码风格
- 添加适当的注释说明功能和参数
- 避免使用 `innerHTML`，使用 `textContent` 或安全的 DOM 操作方法
- 代码缩进使用 2 个空格
- 使用单引号而非双引号

### 翻译规范

- 翻译应准确反映原文含义
- 保持专业术语的一致性
- 考虑中文表达习惯，避免直译
- 对于 GitHub 特有的术语，参考官方中文文档的翻译

### 文档规范

- 使用 Markdown 格式
- 添加适当的标题层级
- 使用表情符号图标增强可读性
- 保持文档内容的简洁明了
- 更新文档时同步更新版本信息

## ❓ 常见问题

### 如何报告问题？
请在 [GitHub Issues](https://github.com/sutchan/GitHub_i18n/issues) 页面提交详细的问题描述，包括复现步骤、浏览器环境和插件版本。

### 如何获取帮助？
可以在 [项目讨论区](https://github.com/sutchan/GitHub_i18n/discussions) 提问，或者参考项目文档和代码注释。

### 如何处理合并冲突？
在推送代码前，先从上游仓库拉取最新代码并解决冲突：
```bash
# 添加上游仓库
git remote add upstream https://github.com/sutchan/GitHub_i18n.git

# 拉取最新代码
git fetch upstream

# 合并到您的分支
git merge upstream/main

# 解决冲突后提交并推送
```
