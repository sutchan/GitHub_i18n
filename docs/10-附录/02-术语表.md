## 📝 命名约定

### 文件命名

- 源文件使用小写字母和连字符（kebab-case）命名
- 配置文件使用小写字母和下划线（snake_case）命名
- 翻译词典文件使用语言代码_国家代码格式（如 zh_CN.js）

### 变量命名

- 变量和函数使用小驼峰命名法（camelCase）
- 常量使用全大写字母和下划线命名
- 类名使用大驼峰命名法（PascalCase）

### 提交信息规范

使用 Conventional Commits 规范，格式为：
```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

类型包括：
- feat: 新功能
- fix: 修复问题
- docs: 文档更新
- style: 格式修改
- refactor: 代码重构
- perf: 性能优化
- test: 测试相关
- build: 构建系统
- ci: CI/CD 相关
- chore: 其他变更
