## 1. 系统要求与安装

### 1.1 浏览器要求

GitHub i18n 插件支持以下浏览器：

- Google Chrome (推荐版本: 80+)  
- Mozilla Firefox (推荐版本: 75+)  
- Microsoft Edge (推荐版本: 80+)  
- Safari (推荐版本: 14+)  

### 1.2 插件安装

#### 1.2.1 通过用户脚本管理器安装

1. **安装用户脚本管理器**
   - Chrome/Firefox/Edge: 安装 [Tampermonkey](https://www.tampermonkey.net/)
   - Safari: 安装 [Userscripts](https://apps.apple.com/us/app/userscripts/id1463298887)

2. **安装 GitHub i18n 插件**
   - 访问 GitHub 插件仓库: [https://github.com/sutchan/GitHub_i18n](https://github.com/sutchan/GitHub_i18n)
   - 点击 `dist` 目录下的 `github_i18n.user.js` 文件
   - 点击 "Raw" 按钮，用户脚本管理器会自动识别并提示安装
   - 点击 "安装" 按钮完成安装

#### 1.2.2 验证安装

1. 安装完成后，打开 GitHub 网站 (https://github.com/)
2. 查看 GitHub 界面是否已自动转换为中文
3. 如果界面未自动转换，可以尝试刷新页面或检查插件是否启用

## 2. 快速开始

### 2.1 首次使用

1. 成功安装插件后，访问任何 GitHub 页面，插件会自动启动并开始翻译
2. 翻译过程中，部分元素可能会先显示英文，然后转为中文，这是正常现象
3. 翻译完成后，可以像平常一样使用 GitHub

### 2.2 基本界面介绍

插件在 GitHub 界面中添加了以下元素：

- **翻译模式切换按钮**: 位于页面顶部工具栏，用于切换翻译模式
- **插件设置面板**: 通过右键点击插件图标或使用快捷键打开
- **翻译状态指示器**: 显示当前页面的翻译进度和状态
- **缓存状态指示器**: 显示当前页面的缓存状态和大小

### 2.3 翻译模式切换

GitHub i18n 插件提供三种翻译模式：

1. **完全翻译**: 将 GitHub 界面上的所有可翻译内容转换为中文
2. **混合模式**: 保留关键术语为英文，其他内容翻译为中文
3. **技术术语优先**: 只翻译界面元素，保留所有技术术语为英文

通过点击顶部工具栏的模式切换按钮，可以在这三种模式之间快速切换。

## 3. 功能操作指南

### 3.1 模式切换操作

**操作步骤**:
1. 访问任何 GitHub 页面
2. 找到页面顶部工具栏中的翻译模式切换按钮
3. 点击按钮打开模式选择菜单
4. 选择需要的翻译模式
5. 页面内容会根据所选模式重新渲染

**效果预览**:
- 选择 "完全翻译" 后，所有内容都会转为中文
- 选择 "混合模式" 后，技术术语保持英文，其他内容转为中文
- 选择 "技术术语优先" 后，只有界面元素会被翻译

### 3.2 自定义翻译设置

**操作步骤**:
1. 点击浏览器插件栏中的 GitHub i18n 图标
2. 选择 "设置" 选项
3. 在设置面板中，可以调整以下选项：
   - 自动翻译触发时机
   - 翻译优先级设置
   - 缓存策略配置
   - 性能优化选项
4. 完成设置后，点击 "保存" 按钮

**高级设置说明**:
- **自动翻译触发时机**: 控制插件何时开始翻译页面内容
- **翻译优先级**: 设置不同类型元素的翻译优先级
- **缓存策略**: 配置多级缓存机制和有效期
- **性能优化**: 调整监控频率、节流延迟、智能DOM监听等参数
- **页面模式适配**: 配置不同页面类型的翻译策略

### 3.3 翻译缓存管理

**操作步骤**:
1. 打开插件设置面板
2. 切换到 "缓存管理" 选项卡
3. 可以执行以下操作：
   - 查看当前缓存统计信息
   - 清除特定页面的翻译缓存
   - 清除所有缓存数据
   - 调整缓存大小限制
4. 点击 "应用" 按钮保存更改

**多级缓存优化建议**:
- 对于频繁访问的页面，建议启用持久化缓存
- 对于翻译更新频繁的页面，可以适当缩短缓存有效期
- 当遇到翻译不准确问题时，尝试清除相关页面的缓存
- 对于大型页面，可以调整内存缓存上限以提高性能
- 对于特殊页面类型（如Codespaces、Copilot），可以启用专用缓存策略

### 3.4 版本更新检查

**操作步骤**:
1. 点击浏览器插件栏中的 GitHub i18n 图标
2. 选择 "检查更新" 选项
3. 插件会自动检查是否有新版本
4. 如果有新版本，会显示更新详情和安装按钮
5. 点击 "安装更新" 按钮完成更新

**自动更新设置**:
1. 打开插件设置面板
2. 切换到 "更新设置" 选项卡
3. 开启 "自动检查更新" 选项
4. 设置检查频率（每日/每周/手动）
5. 选择是否自动安装补丁版本更新
6. 点击 "保存" 按钮

## 4. 配置管理

### 4.1 全局配置

插件的全局配置存储在浏览器的本地存储中，包括：

- 默认翻译模式
- 更新检查频率
- 缓存策略设置
- 性能优化参数
- 用户界面首选项

**重置全局配置**:
1. 打开插件设置面板
2. 切换到 "高级设置" 选项卡
3. 点击 "恢复默认设置" 按钮
4. 在确认对话框中点击 "确定"

### 4.2 页面特定配置

对于不同的 GitHub 页面类型（如仓库主页、Issues 页面、Pull Requests 页面等），插件支持单独的配置设置。

**设置页面特定配置**:
1. 访问目标 GitHub 页面类型
2. 打开插件设置面板
3. 切换到 "页面设置" 选项卡
4. 调整该页面类型的特定设置
5. 点击 "保存" 按钮

### 4.3 导入与导出配置

**导出配置**:
1. 打开插件设置面板
2. 点击 "导入/导出" 选项
3. 点击 "导出配置" 按钮
4. 将生成的配置文件保存到本地

**导入配置**:
1. 打开插件设置面板
2. 点击 "导入/导出" 选项
3. 点击 "导入配置" 按钮
4. 选择之前导出的配置文件
5. 点击 "应用" 按钮完成导入

## 5. 常见场景使用示例

### 5.1 仓库浏览与导航

**场景描述**: 浏览 GitHub 仓库，查看代码、问题和讨论。

**最佳实践**:
1. 使用 "混合模式" 进行翻译，保留技术术语的英文显示
2. 对于不常见的技术术语，可以通过右键菜单查看原始英文
3. 浏览代码文件时，可以使用 "技术术语优先" 模式，确保代码注释正确显示

**使用示例**:
```
1. 打开 GitHub 仓库首页
2. 点击顶部工具栏中的模式切换按钮
3. 选择 "混合模式"
4. 浏览仓库的各个部分，体验本地化的界面
5. 点击 "Code" 标签页查看代码时，技术术语保持英文
```

### 5.2 参与 Issues 讨论

**场景描述**: 参与 GitHub Issues 讨论，回复问题和报告错误。

**最佳实践**:
1. 使用 "完全翻译" 模式，确保讨论内容的完整理解
2. 回复时注意专业术语的正确使用
3. 对于代码片段，考虑切换到 "技术术语优先" 模式查看

**使用示例**:
```
1. 打开 GitHub 仓库的 Issues 页面
2. 点击顶部工具栏中的模式切换按钮
3. 选择 "完全翻译"
4. 浏览和参与讨论，所有文本都将被翻译为中文
5. 需要引用代码时，保持代码块的原始格式
```

### 5.3 Pull Request 工作流

**场景描述**: 查看和审核 Pull Request，参与代码审查。

**最佳实践**:
1. 代码审查时建议使用 "技术术语优先" 模式
2. 阅读描述和评论时，可以切换到 "混合模式"
3. 注意保留代码注释的准确翻译

**使用示例**:
```
1. 打开 Pull Request 页面
2. 首先使用 "混合模式" 阅读描述和讨论
3. 切换到 "技术术语优先" 模式查看代码变更
4. 进行代码审查和评论
5. 提交评论时，注意正确使用专业术语
```

## 6. 快捷键与命令

### 6.1 键盘快捷键

GitHub i18n 插件支持以下键盘快捷键：

| 快捷键 | 功能描述 | 适用范围 |
|--------|---------|----------|
| Alt+T | 切换翻译模式 | 全局 |
| Alt+S | 打开设置面板 | 全局 |
| Alt+R | 刷新页面翻译 | 当前页面 |
| Alt+L | 清除当前页面缓存 | 当前页面 |
| Alt+C | 切换为完全翻译模式 | 全局 |
| Alt+M | 切换为混合模式 | 全局 |
| Alt+T | 切换为技术术语优先模式 | 全局 |

### 6.2 上下文菜单命令

插件在右键菜单中添加了以下命令：

- **翻译选中内容**: 翻译用户选中文本
- **查看原始文本**: 查看已翻译元素的原始英文内容
- **添加到忽略列表**: 将当前元素添加到翻译忽略列表
- **报告翻译问题**: 提交翻译不准确的反馈
- **刷新页面翻译**: 重新翻译当前页面

### 6.3 命令行接口

对于高级用户，插件提供了命令行接口，可以在浏览器控制台中使用：

```javascript
// 切换翻译模式
GitHub_i18n.setPageMode('full'); // 'full', 'mixed', 'technical'

// 刷新页面翻译
GitHub_i18n.refreshTranslation();

// 清除翻译缓存
GitHub_i18n.clearCache();

// 检查更新
GitHub_i18n.checkForUpdates();

// 获取插件信息
GitHub_i18n.getInfo();
```

## 7. 故障排除

### 7.1 常见问题与解决方案

**问题1: 页面未自动翻译**

- **可能原因**: 插件未正确加载或被禁用
- **解决方案**:
  1. 检查浏览器扩展/用户脚本管理器中 GitHub i18n 插件是否启用
  2. 刷新页面尝试重新加载插件
  3. 检查浏览器控制台是否有错误信息
  4. 确保当前 GitHub 页面在支持列表中

**问题2: 部分内容未翻译**

- **可能原因**: 内容在翻译完成后动态加载，或翻译词典不包含该内容
- **解决方案**:
  1. 尝试刷新页面
  2. 使用 Alt+R 快捷键重新翻译页面
  3. 通过右键菜单报告未翻译内容

**问题3: 翻译不准确或错误**

- **可能原因**: 翻译词典需要更新或特定上下文下的翻译不当
- **解决方案**:
  1. 清除当前页面缓存后重试 (Alt+L)
  2. 尝试切换不同的翻译模式
  3. 通过右键菜单报告翻译问题
  4. 等待插件更新，翻译质量会不断改进

**问题4: 插件导致页面性能下降**

- **可能原因**: 大量元素需要翻译，或监控设置过于频繁
- **解决方案**:
  1. 打开设置面板，调整性能优化选项
  2. 降低监控频率和提高节流延迟
  3. 考虑使用 "技术术语优先" 模式，减少翻译工作量
  4. 禁用不必要的页面类型的自动翻译

**问题5: 插件更新失败**

- **可能原因**: 网络问题或用户脚本管理器设置限制
- **解决方案**:
  1. 检查网络连接
  2. 手动访问 GitHub 仓库下载最新版本
  3. 更新用户脚本管理器到最新版本
  4. 尝试在不同浏览器中安装

### 7.2 错误日志与调试

**查看错误日志**:
1. 按 F12 打开浏览器开发者工具
2. 切换到 "Console" 选项卡
3. 查找与 "GitHub i18n" 相关的错误消息
4. 复制错误消息，可以用于报告问题

**启用详细日志**:
1. 打开插件设置面板
2. 切换到 "高级设置" 选项卡
3. 启用 "调试模式"
4. 在浏览器控制台查看详细日志

## 8. 附录

### 8.1 支持的 GitHub 页面类型

GitHub i18n 插件支持以下 GitHub 页面类型：

- 主页和仪表板
- 仓库主页
- Issues 和 Pull Requests
- 代码浏览页面
- Wiki 页面
- 项目看板
- 用户资料页面
- 组织页面
- 活动流
- 设置页面
- GitHub Codespaces
- GitHub Copilot 界面
- GitHub Explore 页面

### 8.2 已知限制

- 实时更新的内容（如实时聊天、实时活动流）可能翻译不完全
- 某些第三方集成组件可能不受支持
- 动态生成的复杂表格可能需要额外刷新才能完全翻译
- 在极端性能受限的环境中，可能会影响页面加载速度
