## 1. 版本号说明

GitHub i18n 插件严格遵循语义化版本规范（SemVer），版本号格式为 `MAJOR.MINOR.PATCH`，其中：

- **MAJOR**: 当进行不兼容的 API 更改或重大架构变更时递增
- **MINOR**: 当添加向下兼容的新功能时递增
- **PATCH**: 当进行向下兼容的问题修复时递增

此外，我们可能会在版本号后附加预发布标识符和构建元数据，例如：

- **预发布版本**: `1.0.0-alpha.1`, `2.0.0-beta.3`
- **构建版本**: `1.2.0+build.20240602`

版本变更类型标签：

- 🆕 **新增**: 新增功能或API
- 🛠️ **改进**: 对现有功能进行改进或优化
- 🐛 **修复**: 修复已知问题或漏洞
- ⚠️ **废弃**: 标记为废弃的功能或API
- ❌ **移除**: 已移除的功能或API
- 🔒 **安全**: 安全相关的修复或增强
- 📝 **文档**: 文档更新
- 🏗️ **架构**: 架构调整或重构

## 5. 重大变更

以下是可能影响现有功能使用的重大变更：

### v1.8.0 重大变更

- **配置存储路径变更**：用户配置的存储路径发生变化，旧版配置会在首次启动时自动迁移。
- **翻译缓存机制更新**：采用了新的缓存策略，可能需要重新加载页面以获取最新翻译。
- **API 响应格式更新**：部分内部 API 的响应格式发生变化，开发者需要更新相关代码。

### v1.7.0 重大变更

- **性能监控机制变更**：新增性能统计功能，可能对初次加载时间有轻微影响。
- **事件系统重构**：事件处理机制全面重构，提高了性能和稳定性。
- **DOM 监听策略调整**：采用新的 DOM 变更检测算法，提高了翻译准确性。

### v1.6.0 重大变更

- **配置验证加强**：对用户配置进行更严格的验证，可能会拒绝无效配置。
- **资源加载优化**：调整了资源加载顺序，可能对某些页面的初始加载速度有影响。
- **错误处理策略更新**：改进了错误恢复机制，在遇到错误时提供更友好的用户体验。

### v1.5.0 重大变更

- **用户界面重构**：配置界面完全重设计，提供更直观的用户体验。
- **存储机制升级**：从 LocalStorage 迁移到 IndexedDB，提高了存储容量和性能。
- **插件权限需求变更**：新增了对某些 API 的权限请求，提高了功能可用性。

### v1.4.0 重大变更

- **翻译算法更新**：采用了改进的翻译匹配算法，提高了翻译准确性。
- **词典结构变更**：词典格式发生变化，首次启动时会自动升级。
- **性能监控增加**：新增了性能指标收集，帮助优化用户体验。

## 6. API 变更历史

### v1.8.x 版本 API 变更

#### translationCore 模块

- **新增**: `translateWithContext(text, context)` - 支持上下文感知的翻译功能
