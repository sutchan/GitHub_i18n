## 1. 目的

本文档为 GitHub 中文翻译插件的开发者提供全面的开发指南，包括环境配置、代码规范、开发流程和贡献方式，帮助开发者快速上手并有效参与项目贡献。

## 2. 概述

本目录包含 GitHub i18n 项目的开发者相关文档，旨在帮助开发者快速了解项目并参与贡献。如果您希望为项目贡献代码、改进功能或修复问题，请仔细阅读本目录下的文档。

## 3. 开发环境设置

在开始开发前，您需要设置适当的开发环境：

- **Node.js** v14+ 和 **npm** v6+
- **Git** 版本控制工具
- 推荐的代码编辑器（如 VS Code）
- 浏览器用户脚本管理器（如 Tampermonkey）用于测试

详细的环境设置步骤请参考[贡献指南](./贡献指南.md)。

## 4. 代码规范

项目遵循以下代码规范：

- 使用 **ES6+** 语法
- 采用小驼峰命名法（camelCase）
- 保持代码缩进和格式一致
- 添加适当的注释，特别是函数级注释
- 文件编码格式为 **UTF-8 无 BOM**
- 换行符为 **Unix LF**

## 5. 开发流程

开发流程通常包括以下步骤：

1. Fork 项目仓库
2. 克隆到本地
3. 创建新分支
4. 实现功能或修复问题
5. 编写测试（如有必要）
6. 提交代码并推送到 GitHub
7. 创建拉取请求

详细的开发流程说明请参考[贡献指南](./贡献指南.md)。

## 6. 贡献指南

我们欢迎各种形式的贡献，包括：

- 代码贡献（功能实现、bug 修复）
- 翻译词典更新
- 文档改进
- 问题报告和建议

请参考[贡献指南](./贡献指南.md)了解如何参与贡献。
