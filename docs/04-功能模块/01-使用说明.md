# 使用说明

## 📋 概述

本文档提供了GitHub中文翻译插件(GitHub_i18n)的详细使用指南，帮助用户了解如何安装、配置和使用该插件，以便在GitHub网站上获得更好的中文用户体验。

## 📥 安装指南

### 1. 前置条件

- 安装支持用户脚本的浏览器扩展：
  - **Chrome/Edge**: [Tampermonkey](https://www.tampermonkey.net/)
  - **Firefox**: [Tampermonkey](https://www.tampermonkey.net/) 或 [Violentmonkey](https://violentmonkey.github.io/)
  - **Safari**: [Userscripts](https://github.com/quoid/userscripts)

### 2. 安装脚本

方法一：从发布页面安装（推荐）
- 访问项目的发布页面
- 点击最新版本的安装链接
- Tampermonkey等扩展会自动弹出安装确认窗口
- 点击「安装」按钮完成安装

方法二：手动安装
- 下载 `GitHub_zh-CN.user.js` 文件
- 打开浏览器扩展管理页面
- 点击「添加新脚本」或「导入」按钮
- 粘贴脚本内容或上传下载的文件
- 保存并启用脚本

## 🚀 使用方法

### 1. 基本使用

安装完成后，插件会自动在访问GitHub网站时激活：

- 插件会自动将GitHub界面中的高频UI元素翻译成中文
- 无需手动触发翻译操作
- 支持GitHub的所有主要页面，包括仓库、Issues、Pull Requests等

### 2. 查看翻译状态

在GitHub页面上，您可以通过以下方式查看翻译状态：

- 翻译后的元素会直接显示中文内容
- 对于尚未翻译的元素，保持原始英文显示
- 页面布局和功能不会受到翻译的影响

### 3. 自动更新

插件内置了自动更新检查功能：

- 每次访问GitHub时，插件会在后台检查是否有新版本
- 发现新版本时，会提示用户进行更新
- 点击更新按钮即可安装最新版本

## ⚙️ 配置选项

### 1. 版本检查设置

插件默认启用自动版本检查，可以通过修改源代码中的配置来调整：

```javascript
// 在CONFIG对象中的相关配置
const CONFIG = {
  // 是否启用自动更新检查
  updateCheck: true,
  // 版本检查的间隔时间（毫秒）
  versionCheckInterval: 24 * 60 * 60 * 1000, // 默认24小时
  // 其他配置...
};
```

### 2. 性能参数调整

高级用户可以调整以下性能相关参数：

- `debounceDelay`: 防抖延迟时间，控制翻译触发频率
- `mutationObserverOptions`: DOM变化监听选项
- `pageMode`: 页面模式设置，影响不同页面类型的翻译策略

## 🛠️ 常见问题

### 1. 插件不生效

- 确认浏览器扩展已正确安装并启用
- 确认脚本已启用
- 尝试刷新GitHub页面
- 检查浏览器控制台是否有错误信息

### 2. 翻译不完整

- 当前版本可能只翻译了部分高频UI元素
- 新发布的GitHub功能可能尚未添加到翻译词典中
- 您可以通过项目的Issues页面报告未翻译的内容

### 3. 页面布局被破坏

- 如果发现页面布局问题，请在GitHub Issues中报告
- 提供问题出现的页面URL和截图
- 临时解决方法：禁用插件并刷新页面

### 4. 性能问题

- 对于大型复杂的GitHub页面，翻译可能会消耗一定的性能
- 您可以调整性能参数来优化体验
- 插件已经实现了节流和缓存机制，以最小化性能影响

## 💡 提示与技巧

- 使用最新版本的插件以获得最佳体验和完整的翻译覆盖
- 定期检查更新以获取新功能和翻译改进
- 报告未翻译的内容有助于改进翻译词典的覆盖范围
- 对于开发者，可以通过修改源代码自定义翻译词典
