## 1. 项目概览

### 1.1 项目定位
GitHub 中文本地化解决方案，提供：
- 轻量级用户脚本（Tampermonkey），用于 GitHub 全站中文翻译
- 自动化字符串抓取与词典更新工具
- 完整的文档体系

### 1.2 项目结构
```
GitHub_i18n/
├── src/                    # 源代码
│   ├── dictionaries/       # 翻译词典
│   ├── config.js           # 配置文件
│   ├── main.js             # 主入口
│   ├── translationCore.js  # 翻译核心
│   ├── pageMonitor.js      # 页面监控
│   └── ...                 # 其他模块
├── utils/                  # 自动化工具
├── docs/                   # 文档目录
├── openspec/               # 项目规范
├── build.js                # 构建脚本
└── README.md               # 项目说明
```

### 2.2 发现的问题

#### 2.2.1 版本不一致
**问题**: 部分源文件版本号不统一
- `src/version.js`: 1.9.1
- `src/main.js`: 1.9.0
- `src/translationCore.js`: 1.8.172
- `src/config.js`: 1.9.0
- `src/virtualDom.js` (推测): 可能不一致

**建议**: 统一所有源文件版本号为 1.9.1

#### 2.2.2 构建系统缺失
**问题**: 项目根目录的 `build.js` 引用了 `build/` 目录下的模块，但该目录不存在
- `build/versionManager.js`
- `build/projectCleaner.js`
- `build/sourceMerger.js`
- `build/buildFixer.js`
- `build/userScriptBuilder.js`
- `build/buildValidator.js`

**影响**: 无法正常执行构建流程

**建议**: 
1. 恢复缺失的构建模块，或
2. 简化构建流程，直接合并源文件

#### 2.2.3 测试缺失
**问题**: 
- 项目根目录无 `package.json`
- `test/` 目录不存在
- 无单元测试、集成测试
- 无 ESLint 配置文件

**建议**:
1. 在根目录添加 `package.json`
2. 建立测试框架（Jest 或 Mocha）
3. 添加 ESLint 配置进行代码规范检查
4. 为核心模块编写单元测试

#### 2.2.4 Trie 树使用问题
**文件**: `src/translationCore.js`
**问题**: 
```javascript
// Line 1467: 调用 findAllMatches 时传递了第二个参数 minKeyLength
const potentialKeys = this.dictionaryTrie.findAllMatches(text, minKeyLength);

// 但 Trie 类的 findAllMatches 方法只接受一个参数（Line 58-96）
```

**影响**: 此功能无法正常工作

**建议**: 修复 Trie 树的 `findAllMatches` 方法，或调整调用方式

#### 2.2.5 version.js 文件问题
**文件**: `src/version.js`
**问题**: 
- 版本历史记录过于冗长，包含大量重复的"自动版本更新"条目
- 最新版本 (1.9.1) 未在 VERSION_HISTORY 中记录

**建议**: 
1. 清理重复的版本历史条目
2. 添加最新版本的变更记录
3. 考虑将完整历史移至单独文件

#### 2.2.6 文档冗余
**问题**: 
- `docs/` 和 `openspec/docs/` 内容重复
- `openspec/docs/_backup/` 包含大量备份文件
- 存在多个 Markdown 格式修复报告

**建议**: 
1. 统一文档目录结构
2. 清理备份和冗余文件
3. 保持单一文档源
