# Markdown格式规范

## 概述
本文档定义了GitHub_i18n项目文档的Markdown格式规范，确保所有文档使用统一的格式和风格。

## 基本规范

### 1. 文件命名
- 使用中文命名，清晰表达文档内容
- 文件扩展名为 `.md`
- 避免使用特殊字符，如 `#`、`?`、`*` 等

### 2. 文件编码
- 编码格式：UTF-8 无 BOM
- 换行符：Unix LF (`\n`)

### 3. 标题层级
```markdown
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（详细内容）
##### 五级标题（补充说明）
###### 六级标题（极少使用）
```

### 4. 标题格式
- 标题与 `#` 符号之间有一个空格
- 标题末尾不加标点符号
- 避免使用空标题
- 每个文档只有一个一级标题

## 内容格式

### 1. 段落
- 段落之间使用空行分隔
- 段落首行不缩进
- 避免过长的段落，建议每段不超过5行

### 2. 列表
#### 无序列表
```markdown
- 列表项1
- 列表项2
  - 子列表项1
  - 子列表项2
```

#### 有序列表
```markdown
1. 列表项1
2. 列表项2
   1. 子列表项1
   2. 子列表项2
```

### 3. 强调
```markdown
*斜体* 或 _斜体_
**粗体** 或 __粗体__
***粗斜体*** 或 ___粗斜体___
```

### 4. 代码
#### 行内代码
```markdown
使用 `console.log()` 输出信息
```

#### 代码块
```javascript
// 代码块示例
function hello() {
  console.log("Hello, world!");
}
```

### 5. 链接
```markdown
[链接文本](URL)
[相对链接](../path/to/file.md)
[带标题的链接](URL "链接标题")
```

### 6. 图片
```markdown
![替代文本](图片URL)
![替代文本](图片URL "图片标题")
```

### 7. 表格
```markdown
| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### 8. 引用
```markdown
> 这是一段引用文本
> 
> 这是引用的第二段
```

### 9. 分隔线
```markdown
---
```

## 特殊元素

### 1. 注释
```markdown

```

### 2. 警告框
```markdown
> **注意**: 这是一个注意事项
> 
> **警告**: 这是一个警告信息
> 
> **提示**: 这是一个提示信息
```

### 3. 任务列表
```markdown
- [x] 已完成任务
- [ ] 未完成任务
```

### 4. 脚注
```markdown
这是一个脚注引用[^1]

[^1]: 这是脚注内容
```

## 最佳实践

1. **保持简洁**: 避免过度复杂的格式和嵌套
2. **语义化**: 使用适当的Markdown元素表达语义
3. **可读性**: 优先考虑文档的可读性而非格式复杂性
4. **一致性**: 在整个项目中保持格式一致性
5. **可维护性**: 使用简单、标准的Markdown语法

## 注意事项

1. 避免使用HTML标签，除非必要
2. 不要使用Markdown扩展语法，除非所有工具都支持
3. 保持文档结构清晰，避免过深的嵌套
4. 定期检查文档格式，确保符合规范
5. 更新文档时，同时更新格式规范
