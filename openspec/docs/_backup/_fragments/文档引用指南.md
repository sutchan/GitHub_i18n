# 文档引用指南

## 概述
本指南说明如何在文档中引用共享内容片段，减少内容重复，提高维护效率。

## 内容片段系统

### 片段目录结构
```
docs/
├── _fragments/           # 内容片段目录
│   ├── 版本信息.md      # 版本信息片段
│   ├── 项目基本信息.md  # 项目基本信息片段
│   └── 技术栈信息.md    # 技术栈信息片段
├── _templates/          # 文档模板目录
│   ├── 基本文档模板.md  # 基本文档模板
│   └── 技术文档模板.md  # 技术文档模板
└── _variables/          # 变量定义目录
    └── 全局变量.md      # 全局变量定义
```

### 可用内容片段

#### 1. 版本信息片段
- **文件路径**: `_fragments/版本信息.md`
- **内容**: 当前版本号、发布日期、版本说明
- **适用场景**: 所有需要显示版本信息的文档

#### 2. 项目基本信息片段
- **文件路径**: `_fragments/项目基本信息.md`
- **内容**: 项目名称、URL、主语言、目标平台等
- **适用场景**: 需要介绍项目基本信息的文档

#### 3. 技术栈信息片段
- **文件路径**: `_fragments/技术栈信息.md`
- **内容**: 核心技术栈、开发工具、浏览器兼容性等
- **适用场景**: 技术文档、开发指南等

## 引用方法

### Markdown引用方式

#### 1. 直接引用
```markdown
## 版本信息

{% include "_fragments/版本信息.md" %}
```

#### 2. 带注释引用
```markdown
<!-- 引用版本信息片段 -->
{% include "_fragments/版本信息.md" %}
```

#### 3. 条件引用
```markdown
{% if page.version %}
{% include "_fragments/版本信息.md" %}
{% endif %}
```

### 相对路径引用

#### 1. 引用同级目录片段
```markdown
{% include "_fragments/版本信息.md" %}
```

#### 2. 引用上级目录片段
```markdown
{% include "../_fragments/版本信息.md" %}
```

#### 3. 引用跨级目录片段
```markdown
{% include "../../_fragments/版本信息.md" %}
```

## 最佳实践

### 1. 引用原则
- 优先使用内容片段，避免重复内容
- 只引用与文档主题相关的内容片段
- 保持引用路径的相对性和一致性

### 2. 片段更新
- 内容片段更新后，所有引用文档自动生效
- 更新片段前，考虑对所有引用文档的影响
- 重大更新时，通知相关文档维护者

### 3. 文档结构
- 将内容片段放在文档的合适位置
- 保持文档结构的逻辑性和连贯性
- 避免过度引用，保持文档的独立性

### 4. 版本控制
- 内容片段的更新纳入版本控制
- 重要片段变更时，记录变更历史
- 定期检查片段内容的有效性

## 注意事项

1. **路径问题**: 确保引用路径正确，建议使用相对路径
2. **循环引用**: 避免内容片段之间相互引用
3. **片段大小**: 保持内容片段的简洁和专注
4. **更新频率**: 考虑内容片段的更新频率和影响范围
5. **兼容性**: 确保引用方式与文档构建系统兼容

## 总结

通过使用内容片段系统，可以：
- 减少文档内容重复
- 提高文档维护效率
- 确保信息一致性
- 简化文档创建流程

建议在创建新文档时，优先考虑使用内容片段，遵循本指南的引用方法和最佳实践。
