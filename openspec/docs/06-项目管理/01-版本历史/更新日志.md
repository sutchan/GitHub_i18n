## 更新日志维护指南

### 添加新版本日志

每次发布新版本时，请按照以下格式添加新的版本日志：

```markdown
### vX.X.X (YYYY-MM-DD)

**版本描述**: 简要描述此版本的主要目的

**主要变更**:

- 🆕 新增功能描述
- 🛠️ 改进功能描述
- 🐛 修复问题描述
- 📝 文档更新描述

**兼容性说明**: 向后兼容性说明
```

### 版本号规范

遵循语义化版本规范（Semantic Versioning）：

- **MAJOR**: 不兼容的 API 更改或重大架构变更时递增
- **MINOR**: 向下兼容的功能性新增时递增
- **PATCH**: 向下兼容的问题修正时递增
